# Personality Prompts

This guide covers how to configure and customize the personality and behavior of your Ravan.ai agents.

## Basic Personality Settings

### 1. Tone and Style

```json
{
  "personality": {
    "tone": "professional",
    "style": "conversational",
    "formality": "context-aware",
    "empathy": "high",
    "enthusiasm": "moderate"
  }
}
```

### 2. Expertise Level

```json
{
  "expertise": {
    "level": "expert",
    "domain": "customer_service",
    "knowledgeDepth": "comprehensive",
    "technicalAccuracy": "high"
  }
}
```

## System Instructions

### 1. Core Behavior

```json
{
  "behavior": {
    "responseStyle": "natural",
    "interruptionHandling": "polite",
    "uncertaintyResponse": "transparent",
    "errorHandling": "graceful"
  }
}
```

### 2. Interaction Rules

```json
{
  "rules": {
    "greeting": "personalized",
    "farewell": "contextual",
    "smallTalk": "minimal",
    "selfDisclosure": "limited"
  }
}
```

## Customization Options

### 1. Language Style

```json
{
  "language": {
    "vocabulary": "professional",
    "sentenceStructure": "varied",
    "idioms": "contextual",
    "culturalReferences": "appropriate"
  }
}
```

### 2. Emotional Intelligence

```json
{
  "emotionalIntelligence": {
    "empathy": "high",
    "activeListening": true,
    "emotionalRecognition": true,
    "responseAppropriateness": "contextual"
  }
}
```

## Best Practices

1. **Personality Design**
   - Match brand voice
   - Consider target audience
   - Maintain consistency
   - Test effectiveness

2. **Behavior Configuration**
   - Set clear boundaries
   - Define response patterns
   - Establish escalation paths
   - Monitor interactions

3. **Testing and Validation**
   - Test with diverse scenarios
   - Gather user feedback
   - Monitor performance
   - Iterate based on results

## Troubleshooting

Common issues and solutions:

1. **Personality Inconsistency**
   - Review configuration
   - Check context handling
   - Verify training data
   - Monitor interactions

2. **Response Quality**
   - Check knowledge base
   - Verify context awareness
   - Monitor user feedback
   - Adjust parameters

3. **User Experience**
   - Gather feedback
   - Analyze interaction patterns
   - Test with real users
   - Iterate based on results
